# تطبيق مرسل رقم المتصل (Call Forwarder)

تطبيق أندرويد بسيط: يراقب المكالمات الواردة، يقرأ رقم المتصل، ويرسله فورًا
عبر شبكة الواي فاي المحلية إلى برنامج "مساعد الدليفري" على الحاسوب.

---

## 🟢 الطريقة بدون حاسوب: بناء APK عبر GitHub من الهاتف

هذا المشروع فيه ملف جاهز (`.github/workflows/build.yml`) يخلي GitHub يبني
لك ملف APK تلقائيًا بالسحابة، وأنت بس ترفع الملفات من متصفح هاتفك.

### الخطوات

1. **افتح github.com من متصفح هاتفك وسوّي حساب** (مجاني)

2. **أنشئ مستودع (Repository) جديد:**
   - اضغط **+ → New repository**
   - اسمه مثلاً `CallForwarder`
   - خله **Public** (يكفي للحساب المجاني)
   - اضغط **Create repository**

3. **ارفع كل ملفات هذا المجلد:**
   - بصفحة المستودع اضغط **Add file → Upload files**
   - اسحب/اختر **كل الملفات والمجلدات** الموجودة هنا (حافظ على نفس بنية
     المجلدات بالضبط: `app/`, `.github/`, `build.gradle`... الخ)
   - ملاحظة: بعض متصفحات الهاتف ما ترفع مجلدات فاضية أو تحافظ على البنية
     تلقائيًا بسهولة — إذا واجهت صعوبة، استخدم تطبيق **GitHub الرسمي**
     (من Google Play) بدل المتصفح، فيه دعم أفضل لرفع الملفات، أو ارفع
     ملفات كل مجلد تدريجيًا (upload files داخل كل مسار على حدة)
   - اكتب رسالة commit بسيطة واضغط **Commit changes**

4. **شغّل البناء (Build):**
   - اذهب لتبويب **Actions** بأعلى صفحة المستودع
   - راح تلاقي "Build APK" شغالة تلقائيًا (لأن الرفع نفسه يشغّلها)
   - إذا ما اشتغلت تلقائيًا: اضغط عليها ثم **Run workflow**
   - انتظر 3-5 دقائق لين تخلص (دائرة خضراء ✅ = نجح، حمراء ❌ = صار خطأ)

5. **نزّل ملف APK:**
   - بعد ما تخلص، افتح الـ workflow run الناجح
   - بالأسفل راح تلاقي قسم **Artifacts** وفيه ملف
     `CallForwarder-debug-apk`
   - اضغط عليه ينزل كملف مضغوط (zip) فيه الـ APK بداخله

6. **ثبّت APK على هاتفك:**
   - فك ضغط الملف (أغلب هواتف أندرويد فيها تطبيق "Files" يفك الضغط تلقائيًا)
   - اضغط على ملف `app-debug.apk`
   - فعّل "السماح من هذا المصدر" (Install unknown apps) لو طلب منك
   - ثبّته

---

## إذا صار خطأ بالبناء (Actions فشلت ❌)

افتح الـ workflow الفاشل وشوف رسالة الخطأ بالأسفل (تكون بالإنجليزية)،
وابعثلي نص الخطأ بالضبط وأساعدك تحلها — أغلب الأخطاء بهالمرحلة تكون بسبب
ملف انرفع بمكان غلط أو ناقص.

---

## بعد التثبيت: صلاحيات لازم تفعلها يدويًا

أندرويد الحديث يحتاج منك تفتح **إعدادات → التطبيقات → CallForwarder → الأذونات**
وتفعل يدويًا:
- الهاتف (Phone / Call logs)
- **إيقاف تحسين البطارية لهذا التطبيق** (Battery optimization → Don't optimize)
  هذا مهم جدًا، خصوصًا بأجهزة شاومي/هواوي/أوبو، وإلا النظام يقفل التطبيق
  بالخلفية ويوقف عن استقبال المكالمات.

## كيف يشتغل

1. تفتح التطبيق أول مرة، تدخل **عنوان IP** الخاص بجهاز الحاسوب على نفس شبكة الواي فاي
   (مثال: `192.168.1.50`) والمنفذ (افتراضيًا `4545`)، وتضغط حفظ.
2. التطبيق يضل شغال بالخلفية.
3. عند أي مكالمة واردة، يرسل رقم المتصل تلقائيًا لجهاز الحاسوب.

## اختبار سريع بدون البرنامج الكامل بعد

بمجلد `pc-server` فيه سيرفر Node.js تجريبي صغير يستقبل الأرقام ويطبعها،
تقدر تستخدمه للتأكد إن الإرسال شغال قبل لا نربطه ببرنامج مساعد الدليفري الكامل.
هذا الجزء بس يحتاج حاسوب لأنه هو نفسه البرنامج المطلوب تشغيله لاحقًا.

## ملاحظة مهمة

رقم المتصل (`EXTRA_INCOMING_NUMBER`) بعض الأجهزة تحجبه إلا إذا صلاحية
`READ_CALL_LOG` مفعّلة إضافة لصلاحية `READ_PHONE_STATE` — الاثنتين مطلوبتين
بالمانيفست وبالكود (موجودتين مسبقًا بهذا المشروع).

## بديل أسرع بدون بناء أي شي: MacroDroid

لو صعبت عليك خطوات GitHub، تذكر إنه فيه بديل أبسط بكثير — تطبيق **MacroDroid**
الجاهز من المتجر يسوي نفس الوظيفة بدون أي بناء أو كود، بس تعريف "ماكرو" واحد
بالتطبيق نفسه (اتصال وارد → إرسال طلب HTTP).
